# 07 前端实现

前面的章节我们已经把API都实现好了，并且通过Rest Client测试过了。接下来我们就可以实现UI界面了。

回顾[04 API实现](./04%20API实现.MD)开头我们介绍App Router介绍
```
页面文件位于app/目录，如app/about/page.tsx对应/about路径，支持嵌套布局（如app/dashboard/layout.tsx）。
```

所以应用的默认入口的页面就是/app/page.tsx，同时该页面也默认会套用/app/layout.tsx。

当我们调用create-next-app创建应用的时候，就帮我们生成了这两个页面。

接下来我们基于ant-design来实现前端页面。

## 7.1 安装ant-design
```
npm i @ant-design/icons antd -S
```

## 7.2 页面实现

前面我们提到系统有一个默认的layout.tsx，不特别声明的话，所有页面都会套用这个layout。

这种后台管理页面我们第一反应就会想到用一个"广"字型布局，Header显示logo、当前登录用户信息等，左侧显示菜单栏，右下角作为内容区。

但是我们还有登录页面，不应该套用"广"字型布局。我们可以改一下layout.tsx文件，如果是login页面就返回空。
